## Usage Patterns

### Flutter Feature Development
```
1. flutter-explorer → Understand existing patterns
2. flutter-architect → Design solution
3. User approves
4. flutter-implementer → Build feature
5. Specialists as needed (animator, platform-integrator, etc.)
6. flutter-tester → Write tests
7. flutter-performance-optimizer → Optimize
8. flutter-reviewer → Final review
```

### React Feature Development

```
1. react-explorer → Understand codebase
2. react-architect → Design components
3. User approves
4. react-implementer → Build feature

6. react-tester → Write tests
7. react-performance-optimizer → Optimize
8. react-reviewer → Final review
```

## Multi-Agent Workflows

### Example: Flutter App with Native Integration

```
Phase 1: Exploration (parallel)
├─ flutter-explorer → App patterns
└─ flutter-platform-integrator → Existing platform code

Phase 2: Design
└─ flutter-architect → Overall design

Phase 3: Implementation (parallel)
├─ flutter-implementer → Dart/Flutter code
├─ kotlin-expert → Android native
└─ swift-expert → iOS native

Phase 4: Integration
└─ flutter-platform-integrator → Wire everything

Phase 5: Testing
└─ flutter-tester → Comprehensive tests

Phase 6: Review
└─ flutter-reviewer → Final check
```

### Example: Tauri Desktop App with Rust Backend

```
Phase 1: Exploration (parallel)
├─ tauri-explorer → Existing commands, IPC patterns
└─ rust-expert → Existing Rust modules

Phase 2: Design
└─ tauri-architect → Command design, IPC flow, frontend integration

Phase 3: Implementation (parallel)
├─ tauri-implementer → Rust commands + frontend
└─ rust-expert → Complex async Tokio operations

Phase 4: Security Review
└─ tauri-security-specialist → CSP, capabilities, path traversal

Phase 5: Testing (parallel)
├─ tauri-tester → Rust unit tests + E2E
└─ rust-expert → Integration tests

Phase 6: Review
└─ tauri-reviewer → Final check
```

### Example: Vue.js App with SSR

```
Phase 1: Exploration
└─ vue-explorer → Existing composables, Pinia stores

Phase 2: Design (parallel)
├─ vue-architect → Component structure, composables
└─ vue-ssr-specialist → SSR strategy, Nuxt.js patterns

Phase 3: Implementation
└─ vue-implementer → Build with Composition API

Phase 4: SSR Implementation
└─ vue-ssr-specialist → Server composables, data fetching

Phase 5: Testing
└─ vue-tester → Vitest component tests + Playwright E2E

Phase 6: Optimization & Review (parallel)
├─ vue-performance-optimizer → Reactivity optimization
└─ vue-reviewer → Final check
```

### Example: Svelte App with SvelteKit

```
Phase 1: Exploration
└─ svelte-explorer → Existing runes patterns, stores

Phase 2: Design (parallel)
├─ svelte-architect → Component structure, routing
└─ sveltekit-specialist → SSR/SSG strategy, adapters

Phase 3: Implementation
└─ svelte-implementer → Build with runes ($state, $derived, $effect)

Phase 4: SvelteKit Integration
└─ sveltekit-specialist → Load functions, form actions

Phase 5: Testing
└─ svelte-tester → Vitest + Playwright tests

Phase 6: Optimization & Review (parallel)
├─ svelte-performance-optimizer → Bundle optimization
├─ svelte-compiler-specialist → Compiler output analysis
└─ svelte-reviewer → Final check
```

### Example: React Native Mobile App

```
Phase 1: Exploration (parallel)
├─ react-native-explorer → Existing navigation, state
└─ react-native-platform-specialist → Platform-specific code

Phase 2: Design
└─ react-native-architect → Navigation flow, state management

Phase 3: Implementation (parallel)
├─ react-native-implementer → Core feature (iOS/Android)
└─ react-native-bridge-specialist → Native modules if needed

Phase 4: Platform Features (parallel)
├─ react-native-platform-specialist → iOS-specific features
└─ react-native-platform-specialist → Android-specific features

Phase 5: Testing
└─ react-native-tester → Jest unit tests + Detox E2E

Phase 6: Optimization & Review (parallel)
├─ react-native-performance-optimizer → Rendering, memory
└─ react-native-reviewer → Final check
```

## Key Features

- **Specialized Expertise**: Each agent has deep domain knowledge
- **Parallel Execution**: Run multiple agents simultaneously
- **Language Experts**: Cross-framework Dart/TS/JS/Kotlin/Swift/Rust expertise
- **Framework Specific**: Tailored agents for Flutter, React, Tauri, Vue.js, Svelte, and React Native
- **Complete Workflows**: From exploration to review across all frameworks
- **Quality Focus**: Testing and review built-in for every framework
- **Security First**: Specialized security agents for Tauri and React Native
- **Performance Optimization**: Dedicated performance agents for each framework
- **Accessibility**: WCAG compliance agents for all UI frameworks
